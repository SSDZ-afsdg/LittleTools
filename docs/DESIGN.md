## 3. 手机端模块

### 3.1 输入模块

| 项目 | 内容 |
|---|---|
| **来源需求** | 需求文档 2.1、2.2 |
| **职责** | 接收用户输入，执行本地关键词分类 |
| **具体功能** | ① 语音输入：调用 Android 系统语音识别（Web Speech API）<br>② 文字输入：文本框手动输入<br>③ 本地关键词匹配：快速分类，不依赖 AI<br>④ 生成记录对象，交给存储模块 |
| **输入** | 用户语音 / 用户文字 |
| **输出** | 结构化记录对象（含本地分类结果） |
| **备注** | 语音转文字方案选用 Android 系统语音识别，不引入 Whisper |


### 3.2 存储模块（手机端）

| 项目 | 内容 |
|---|---|
| **来源需求** | 需求文档 6、7 |
| **职责** | 记录在手机端的增删改查 + 状态管理 |
| **具体功能** | ① 新增记录：存入 IndexedDB，状态为 `draft`<br>② 修改记录：更新 IndexedDB，**修改后自动转为 `pending`**<br>③ 删除记录：软删除<br>④ 查询记录列表<br>⑤ 状态变更：`draft` → `pending`（用户确认后）<br>⑥ 同步成功后：`pending` → `synced` |
| **输入** | 记录对象 / 记录ID + 修改内容 |
| **输出** | 操作结果 / 记录列表 |
| **备注** | 手机端存储方案为 IndexedDB，电脑端存储方案为 MySQL，已确认 |


### 3.3 流水模块

| 项目 | 内容 |
|---|---|
| **来源需求** | 需求文档 5.1 |
| **职责** | 展示记录列表，提供修改/删除入口 |
| **具体功能** | ① 按时间倒序列出所有记录<br>② 每条记录显示：金额/类别/描述/日期<br>③ 点击记录进入编辑<br>④ 左滑/长按提供删除入口<br>⑤ 按状态筛选（全部/草稿/待同步/已同步） |
| **输入** | 用户点击/滑动操作 |
| **输出** | 编辑界面 / 删除确认弹窗 |


### 3.4 分析模块（手机端）

| 项目 | 内容 |
|---|---|
| **来源需求** | 需求文档 5.1 |
| **职责** | 展示同步后拉取的月度报告（简版） |
| **具体功能** | ① 展示简版报告：总支出 + 类别占比饼图 + AI 总结文字<br>② 数据来源：同步时电脑端实时返回<br>③ 缓存策略：缓存最近一次同步结果，离线可查看 |
| **输入** | 同步时电脑端返回的分析数据 |
| **输出** | 简版月度报告（图表 + 文字） |
| **备注** | 手机端分析模块只做展示，不自行计算 |


### 3.5 设置模块

| 项目 | 内容 |
|---|---|
| **来源需求** | 需求文档 5.1 |
| **职责** | 提供系统配置和手动操作入口 |
| **具体功能** | ① 手动同步按钮<br>② 显示待同步记录数量<br>③ 显示同步状态（成功/失败/冲突）<br>④ 关于信息 |
| **输入** | 用户点击按钮 |
| **输出** | 同步执行 / 状态反馈 |


## 4. 电脑端模块

### 4.1 同步服务模块

| 项目 | 内容 |
|---|---|
| **来源需求** | 需求文档 3.2 |
| **职责** | 接收手机端的同步请求，处理数据入库，实时返回分析结果 |
| **具体功能** | ① 接收 HTTP POST /sync 请求<br>② 解析 JSON 数据<br>③ 逐条调用解析模块（AI 重新分类）<br>④ 检查冲突（基于 `version` 字段）<br>⑤ 如有冲突：整个同步失败，一条也不更新，返回冲突列表<br>⑥ 无冲突：存入 MySQL 主库<br>⑦ 调用统计模块生成分析数据<br>⑧ 返回同步结果 + 分析数据给手机端 |
| **输入** | HTTP 请求 + JSON 数据 |
| **输出** | HTTP 响应（同步结果 + 分析数据） |
| **备注** | 同步采用"全有或全无"策略：有冲突则全部失败 |


### 4.2 解析模块（Ollama）

| 项目 | 内容 |
|---|---|
| **来源需求** | 需求文档 2.3 |
| **职责** | 调用 Ollama 对原始文本进行语义解析，提取结构化信息 |
| **具体功能** | ① 接收原始文本<br>② 构造 Prompt<br>③ 调用 Ollama API<br>④ 解析返回结果，提取金额/类别/描述 |
| **输入** | 原始文本（raw_text） |
| **输出** | { amount, category, description } |
| **异常处理** | 解析失败时直接报错，本次同步失败，用户稍后重试 |
| **备注** | 不做兜底，失败即报错 |


### 4.3 统计模块

| 项目 | 内容 |
|---|---|
| **来源需求** | 需求文档 2.4、2.5、2.6、2.8 |
| **职责** | 对数据进行汇总计算，输出结构化数据 |
| **具体功能** | ① 月度总支出 / 笔数 / 日均<br>② 类别分布<br>③ 每日趋势<br>④ 与上月对比<br>⑤ 最大单笔消费 |
| **输入** | 年份 + 月份 |
| **输出** | 结构化统计数据（供报告生成模块使用） |
| **备注** | 只输出数据，不负责格式化/展示 |


### 4.4 报告生成模块（Jupyter Notebook）

| 项目 | 内容 |
|---|---|
| **来源需求** | 需求文档 5.2 |
| **职责** | 生成一页式月度报告 |
| **具体功能** | ① 读取统计模块的数据<br>② 生成总览卡片<br>③ 生成类别分布饼图（matplotlib）<br>④ 生成每日趋势折线图（matplotlib）<br>⑤ 生成与上月对比<br>⑥ 调用 Ollama 生成月度总结<br>⑦ 生成消费建议<br>⑧ 输出为 HTML 文件 |
| **输入** | 年份 + 月份（默认当前月） |
| **输出** | HTML 文件（浏览器打开查看） |
| **备注** | 输出形式为 HTML 文件，非 Jupyter 内联渲染 |


### 4.5 导出模块

| 项目 | 内容 |
|---|---|
| **来源需求** | 需求文档 2.12 |
| **职责** | 导出年度数据，支持归档 |
| **具体功能** | ① 导出 CSV（Excel 可打开）<br>② 导出 JSON（可导入其他系统）<br>③ 导出指定年份<br>④ 导出后提醒用户确认，然后清理数据 |
| **输入** | 年份 |
| **输出** | 文件（.csv + .json） |
| **运行方式** | 在 Jupyter Notebook 中手动运行 Cell 执行 |

## 6. 核心数据流

### 6.1 日常录入（手机端）
用户语音/文字
│
▼
输入模块：本地关键词匹配
│
▼
存储模块：存入 IndexedDB（status: draft）
│
▼
用户修改 → 自动转为 pending
用户确认 → 转为 pending

### 6.2 同步流程（完整闭环）
手机端点击"同步"
│
▼
发送所有 pending 记录 → 电脑端 /sync
│
▼
电脑端逐条处理：
① 调用 Ollama 解析
② 检查冲突（version 字段）
│
├── 有冲突 → 整个同步失败，返回冲突列表 → 手机端提示"同步失败，请处理冲突"
│
└── 无冲突 → 存入 MySQL
│
▼
③ 调用统计模块 → 生成分析数据
④ 调用 Ollama → 生成月度总结文字
│
▼
返回给手机端：

同步结果（成功/冲突）

分析数据（总支出/类别分布/趋势/AI总结）
│
▼
手机端：
① 标记已同步记录为 synced
② 缓存分析数据到 IndexedDB
③ 显示"同步完成"
④ 分析模块展示最新报告

### 6.3 查看分析（手机端）
用户打开"分析"Tab
│
▼
检查缓存中是否有数据
│
┌───┴───┐
▼ ▼
有缓存 无缓存
│ │
▼ ▼
显示缓存 显示"请先同步"
（提示是否最新）


## 7. 技术栈确认

| 层级 | 选型 | 说明 |
|---|---|---|
| **手机端框架** | PWA（HTML + CSS + JS） | 可添加到主屏幕 |
| **手机端存储** | IndexedDB | 离线暂存 |
| **手机端语音** | Web Speech API | Android 系统语音识别 |
| **电脑端** | Jupyter Notebook + FastAPI | 分析 + API |
| **电脑端 AI** | Ollama（qwen2.5:7b） | 本地运行 |
| **电脑端存储** | MySQL | 主数据库 |
| **可视化** | matplotlib | 生成图表 |
| **同步协议** | HTTP REST API | 局域网通信 |


## 8. 数据库表结构（概要）

| 字段 | 类型 | 说明 |
|---|---|---|
| `id` | VARCHAR(36) | 主键，UUID |
| `raw_text` | VARCHAR(500) | 用户原始输入 |
| `amount` | DECIMAL(10,2) | 金额 |
| `category` | VARCHAR(50) | 分类 |
| `description` | VARCHAR(200) | 简短描述 |
| `date` | DATETIME | 消费时间 |
| `created_at` | DATETIME | 录入时间 |
| `synced_at` | DATETIME | 同步时间 |
| `ai_parsed` | JSON | AI 解析结果 |
| `is_deleted` | TINYINT | 软删除标记 |
| `version` | INT | 冲突检测版本号 |

> 详细 DDL 见详细设计文档


## 9. 模块依赖关系
手机端：
输入模块 → 存储模块
存储模块 → 流水模块 / 分析模块 / 设置模块
设置模块 → 同步服务（HTTP）
同步服务 → 分析模块（数据写入）

电脑端：
同步服务 → 解析模块 → 存储模块（MySQL）
同步服务 → 冲突检测 → 存储模块
同步服务 → 统计模块 → 报告生成模块
报告生成模块 → 导出模块（独立）
