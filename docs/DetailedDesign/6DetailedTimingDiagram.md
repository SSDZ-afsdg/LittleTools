## 6. 详细时序图（核心场景）


### 6.1 场景一：用户录入 + 本地存储（手机端）

```
用户                输入模块              存储模块             IndexedDB
 │                    │                    │                    │
 │  ① 说"吃火锅186"   │                    │                    │
 │───────────────────▶│                    │                    │
 │  或输入文字         │                    │                    │
 │                    │                    │                    │
 │                    │  ② 本地关键词匹配   │                    │
 │                    │  金额:186          │                    │
 │                    │  类别:餐饮         │                    │
 │                    │  描述:吃火锅       │                    │
 │                    │                    │                    │
 │                    │  ③ 生成记录对象    │                    │
 │                    │  status: 'draft'   │                    │
 │                    │                    │                    │
 │                    │  ④ 调用添加        │                    │
 │                    │───────────────────▶│                    │
 │                    │                    │                    │
 │                    │                    │  ⑤ 存入            │
 │                    │                    │───────────────────▶│
 │                    │                    │                    │
 │                    │                    │  ⑥ 返回成功        │
 │                    │                    │◀───────────────────│
 │                    │                    │                    │
 │  ⑦ 提示"已记录"    │  ⑧ 返回           │                    │
 │◀───────────────────│◀───────────────────│                    │
 │                    │                    │                    │
```

**关键点**：
- 用户录入后**不需要手动确认**，直接存入 `draft` 状态
- 本地关键词匹配在 0.1 秒内完成
- 用户修改记录时，状态自动转为 `pending`

```

     │────────────▶│                │                │              │             │
     │              │                │                │              │             │

     │              │───────────────────────────────────────────────────────────────▶│
     │              │                │                │              │             │
     │              │  ③ 返回记录列表│                │              │             │
     │              │◀───────────────────────────────────────────────────────────────│
     │              │                │                │              │             │
     │              │  ④ POST /sync │                │              │             │
     │              │────────────────▶│                │              │             │
     │              │                │                │              │             │
     │              │                │  ⑤ 逐条解析   │              │             │
     │              │                │───────────────▶│              │             │
     │              │                │                │              │             │

     │              │                │◀───────────────│              │             │

     │              │                │  ⑦ 版本冲突检测│              │             │

     │              │                │  ⑧ 检测到冲突 │              │             │

     │              │                │                │              │             │

     │              │                │  (Ollama)     │              │             │
```

**关键点**：
- 有冲突时**整个同步失败**，一条也不更新
- 无冲突时**实时计算分析数据**，同步完即可查看
- 分析数据**缓存到 IndexedDB**，离线可看


### 6.3 场景三：手机端查看分析报告

```
用户           分析模块(Tab)      缓存(IndexedDB)     同步模块
  │                  │                  │               │
  │  ① 切到"分析"Tab │                  │               │
  │─────────────────▶│                  │               │
  │                  │                  │               │
  │                  │  ② 查询缓存     │               │
  │                  │─────────────────▶│               │
  │                  │                  │               │
  │                  │  ③ 返回缓存数据 │               │
  │                  │◀─────────────────│               │
  │                  │                  │               │
  │  ┌───────────────┴──────────────────┴───────────────┘
  │  │ 判断：缓存是否有效？
  │  │ - 是否有数据？
  │  │ - 是否是当前月？
  │  │ - 是否过期？（超过1天）
  │  └─────────────────────────────────────────────────────
  │                  │                  │               │
  │   如果有有效缓存： │                  │               │
  │  ④ 显示简版报告  │                  │               │
  │   - 总支出       │                  │               │
  │   - 饼图         │                  │               │
  │   - AI总结       │                  │               │
  │◀────────────────▶│                  │               │
  │                  │                  │               │
  │   如果无缓存/过期：│                  │               │
  │  ⑤ 显示         │                  │               │
  │   "暂无数据，   │                  │               │
  │    请先同步"    │                  │               │
  │◀────────────────▶│                  │               │
  │                  │                  │               │
```

**关键点**：
- 分析 Tab 优先读缓存，不实时请求电脑（避免每次打开都要联网）
- 缓存附带有数据时效性提示（如"数据来自 7月6日"）
- 同步后自动更新缓存


### 6.4 场景四：冲突处理（电脑端 Jupyter）

```
用户                     Jupyter Notebook                    MySQL
  │                              │                             │
  │  ① 运行冲突检测Cell          │                             │
  │─────────────────────────────▶│                             │
  │                              │                             │
  │                              │  ② 查询所有冲突记录        │
  │                              │  (version不匹配的记录)      │
  │                              │────────────────────────────▶│
  │                              │                             │
  │                              │  ③ 返回冲突列表            │
  │                              │◀────────────────────────────│
  │                              │                             │
  │  ④ 显示冲突对比              │                             │
  │  ┌───────────────────────┐   │                             │
  │  │ 记录ID: xxx           │   │                             │

  │  │ [使用手机] [使用电脑]  │   │                             │
  │  └───────────────────────┘   │                             │
  │◀─────────────────────────────│                             │
  │                              │                             │
  │  ⑤ 用户选择使用哪个版本      │                             │
  │─────────────────────────────▶│                             │
  │                              │                             │
  │                              │  ⑥ 更新数据库              │
  │                              │  (用选中的版本覆盖)         │
  │                              │────────────────────────────▶│
  │                              │                             │
  │  ⑦ 显示"冲突已解决"         │  ⑧ 返回成功                │
  │◀─────────────────────────────│◀────────────────────────────│
  │                              │                             │
```

**关键点**：
- 冲突处理在电脑端 Jupyter 中手动完成
- 解决后重新同步即可（手机端该记录会变为 `synced`）
- 整个设计保证了数据一致性，用户掌握最终决定权
